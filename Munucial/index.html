Beginner or Professional 		Why called Munucial? It's because while I'm building this project I want to be a high quality, multi-function CMS or web application, so this project called Munucial. Munucaial is a combine of munus which is function in Latin and suffix -cial which means quality.

Why I want to build Munucial

Why would I want to build Munucial
or a self-made CMS?

it's because when you're making a website, different system have different feature. I get in a case very often when you're using others' CMS, that is after you use it for a while you found out the it does not contain some feature that is really imortant to you, or it's performance (like load speed or security) isn't just as well as you expect. And also there is a restriction, most of the CMS use PHP for routing and that will make a restriction for accessing you're own data, like if you upload your file to a path you can't really accessing them by typing the path in the address bar. So I want to build my own customized CMS or I called it multi-function (all-in-one) CMS.

How do we serve contents

We support serving content in many ways, as we mention above most CMS will have restriction on accessing your own data, but we don't.
We have a flow that follows our serving priority.

  1. Posts that save content in JSON db
  2. Posts that save content in HTML file
  3. HTML files (including system's and yours)
  4. Javascript/CSS (JS first than CSS) files (system's & yours)
  5. All the other files

Also we have <pre> function which let you view the code. more info
Notice: any file with extensionare other files, html/js/css files' priority are serve by non-extension URL

We serve content as first priority from JSON database which was store in the following structure
